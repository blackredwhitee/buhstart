# Сайт «Доверительная Бухгалтерия»

Готовый статический сайт лежит в папке **`docs/`** — это обычные HTML/CSS/JS файлы,
никакой сборки для запуска не нужно.

## Как открыть локально

Просто откройте `docs/index.html` в браузере — или запустите простой сервер:

```
cd docs
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

## Как опубликовать на GitHub Pages

1. Закоммитьте и запушьте репозиторий на GitHub.
2. В репозитории откройте **Settings → Pages**.
3. В разделе **Build and deployment → Source** выберите **Deploy from a branch**.
4. Ветка — `main`, папка — **`/docs`**. Нажмите **Save**.
5. Через 1–2 минуты сайт откроется по адресу `https://<ваш-логин>.github.io/<репозиторий>/`.

Главная страница — `index.html`, поэтому корневой адрес сразу покажет сайт.

## Страницы

`index` · `uslugi` · `usluga` (детальная услуга, `usluga.html?s=strahovka` и т.п.) ·
`calculator` · `blog` · `article` · `novosti` · `team` · `contacts` ·
`privacy` · `soglasie`.

## Что важно знать

- **Формы — демонстрационные.** Кнопки показывают «Отправлено», но данные никуда не
  уходят. Чтобы принимать реальные заявки, нужен сервис приёма форм (Formspree,
  Telegram-бот и т.п.).
- **Фото сотрудников и команды** пока показаны заглушками — замените их реальными
  снимками (см. `docs/assets/` и разметку страниц).
- Виджеты Яндекса (отзывы на главной, карта на «Контактах») подгружаются с yandex.ru.

## Как менять содержимое

- **Текст/структура страниц, общая шапка и подвал** — в `tools/build.js`. После правок
  выполните `node tools/build.js`, и файлы в `docs/` перегенерируются.
- **Стили** — `docs/assets/styles.css`. **Поведение** (меню, калькулятор, формы) —
  `docs/assets/main.js`. Их правят напрямую.
